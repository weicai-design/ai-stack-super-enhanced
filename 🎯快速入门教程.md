## 🎓 15分钟完整流程

### 场景: 完整的订单到交付流程

#### 第1步: 创建客户和订单（3分钟）

```python
# 1. 创建客户
POST http://localhost:8013/api/erp/customer/create
{
  "customer_id": "CUST001",
  "name": "ABC公司",
  "industry": "电子制造"
}

# 2. 创建订单
POST http://localhost:8013/api/erp/order/create
{
  "customer_id": "CUST001",
  "items": [
    {"product_id": "PROD001", "quantity": 1000, "price": 100}
  ]
}
```

#### 第2步: 采购物料（4分钟）

```python
# 3. 创建采购申请
POST http://localhost:8013/api/erp/procurement/request/create
{
  "requester": "张三",
  "items": [
    {"material_id": "MAT001", "quantity": 500, "spec": "标准规格"}
  ],
  "reason": "生产订单CUST001",
  "required_date": "2025-11-10"
}

# 4. 审批采购申请（会自动生成采购订单）
POST http://localhost:8013/api/erp/procurement/request/approve
{
  "request_id": "PR20251106XXXX",
  "approver": "经理",
  "approved": true
}

# 5. 记录到货
POST http://localhost:8013/api/erp/receipt/arrival/record
{
  "receipt_id": "RN20251106XXXX",
  "actual_items": [
    {"material_id": "MAT001", "quantity": 500}
  ],
  "received_by": "仓管员",
  "arrival_date": "2025-11-08"
}
```

#### 第3步: 生产和质检（3分钟）

```python
# 6. 创建生产计划
POST http://localhost:8013/api/production/plan/create
{
  "order_id": "ORD20251106XXXX",
  "product_id": "PROD001",
  "quantity": 1000,
  "start_date": "2025-11-09"
}

# 7. 质量检验
POST http://localhost:8013/api/quality/inspection/create
{
  "batch_id": "BATCH001",
  "inspector": "质检员"
}
```

#### 第4步: 交付和发运（3分钟）

```python
# 8. 创建交付计划
POST http://localhost:8013/api/erp/delivery/plan/create
{
  "sales_order_id": "ORD20251106XXXX",
  "customer_id": "CUST001",
  "delivery_date": "2025-11-15",
  "items": [
    {"material_id": "PROD001", "quantity": 1000}
  ]
}

# 9. 创建发运计划
POST http://localhost:8013/api/erp/shipping/plan/create
{
  "delivery_orders": ["DO20251106XXXX"],
  "destination": "客户地址",
  "transport_mode": "truck",
  "planned_date": "2025-11-15"
}
```

#### 第5步: 监控和分析（2分钟）

```python
# 10. 查看ERP总览
GET http://localhost:8013/api/erp/dashboard/overview

# 11. 查看关键指标
GET http://localhost:8013/api/erp/dashboard/kpi

# 12. 查看闭环监控
GET http://localhost:8013/api/erp/monitoring/dashboard
```

**完成！** 🎉 你已经走完了一个完整的订单到交付流程！

## ❓ 常见问题

### Q1: 服务启动失败怎么办？

**A**: 检查日志文件
```bash
cat logs/[服务名].log
```

常见原因:
- 端口被占用 → 修改端口或停止占用进程
- 依赖缺失 → `pip install -r requirements.txt`
- 配置错误 → 检查 `.env` 文件

### Q2: AI回复速度慢怎么办？

**A**: 检查配置
```bash
# 1. 确保使用本地模型（Ollama）
# 2. 检查网络连接
# 3. 查看缓存配置

# 优化建议
# 启用缓存: ENABLE_CACHE=true
# 减少上下文: MAX_CONTEXT_LENGTH=2000
```

### Q3: 如何添加新的知识？

**A**: 三种方式
1. **拖拽上传**: 直接拖到聊天框
2. **API上传**: `POST /api/rag/upload`
3. **命令上传**: `@rag upload 文件路径`

### Q4: 如何备份数据？

**A**: 使用备份脚本
```bash
python3 scripts/backup_restore.py --backup
```

### Q5: 如何自定义配置？

**A**: 编辑 `.env` 文件
```bash
# 复制示例配置
cp env.example .env

# 编辑配置
nano .env

# 重启服务
./scripts/quick_start.sh
```
