## 一、RAG和知识图谱功能

| 功能名称 | 具体功能要求 |
|---------|-------------|
| 全格式文件处理 | 支持所有办公文件、电子书、编程文件、图片、语音、视频、思维导图、数据库文件和其他文本文件的上传、分块、解析、检索。必须是**所有格式文件** |
| 四项预处理 | 所有文件、数据进入RAG库前，进行四项预处理（清洗、标准化、去重、验证） |
| 真实性验证 | 所有进入RAG库的信息都会进行去伪处理，保证信息知识数据等的真实性和准确性 |
| 多源信息收集 | 聊天搜索到的网络信息、数据、人机交互信息和其他智能体产生的业务信息，将有用、有效信息经过预处理后进入RAG库 |
| 知识检索利用 | 聊天、各种智能体都会检索、利用RAG库的知识、信息、数据 |
| 自主分组 | RAG库的内容会根据词义进行自主分组 |
| 前后端实现 | RAG前端功能实现操作界面，后端与各智能体、企业管理功能、AI-Stack所有其他功能关联且相互调用 |
| 知识图谱构建 | RAG所有形成的数据、知识、信息最终形成知识图谱 |
| Web界面 | RAG和知识图谱的各个功能形成前端操作界面，有网址、端口并在Docker上运行 |

## 二、企业经营、运营、管理功能

### 2.1 经营管理

| 功能名称 | 具体功能要求 |
|---------|-------------|
| 财务数据管理 | 投入、产出、收入、负债、资产、利润的数据导入导出 |
| 自主财务看板 | 日、周、月、季、年的自主财务看板 |
| 盈亏平衡分析 | 按照"利润=产出-投入"进行盈亏平衡分析 |
| 经营诊断 | 对开源问题、费用支出问题、成本投入问题、产出效率、运营效率等进行分析、诊断 |
| 经营建议 | 参考RAG、行业数据提供经营指导意见 |

#### 开源分析

| 功能名称 | 具体功能要求 |
|---------|-------------|
| 客户订单分析 | 客户类别、订单量、长期与短期订单、订单单价分析 |
| 项目影响分析 | 新开发项目的潜在订单对各月、各季度、全年、三年、五年销售额的影响 |
| 行业对比分析 | 结合同行业分析对未来发展的影响，对公司战略的影响 |
| 订单明细分析 | 订单明细、日/周/月/季/年/三年的分类汇总分析 |
| 关键因素识别 | 找出对"利润=产出-投入"的各项关键影响因素 |

#### 成本分析

| 功能名称 | 具体功能要求 |
|---------|-------------|
| 费用合理性分析 | 销售费用、财务费用、管理费用、生产人工、制造费用、材料费用相对于订单、经营计划、利润计划的合理性 |
| 盈亏平衡影响 | 这些费用综合起来对于盈亏平衡的影响 |
| 价值分析 | 发生或支出的价值分析 |
| 成本明细分析 | 费用、成本的明细、日/周/月/季/年/三年的分类汇总分析 |
| 关键因素识别 | 找出对"利润=产出-投入"的各项关键影响因素 |

#### 产出效益

| 功能名称 | 具体功能要求 |
|---------|-------------|
| 投入回报分析 | 所有投入成本、支出费用产生的回报、效益、效率 |
| 明细到汇总 | 从具体明细，到日/周/月/季/年/三年的分类汇总分析 |
| 关键因素识别 | 找出对"利润=产出-投入"的各项关键影响因素 |

### 2.2 运营与管理

| 功能名称 | 具体功能要求 |
|---------|-------------|
| 流程定义 | 制定流程名、流程环节、流程作业内容、流程跟踪、流程改进、流程KPI |
| 扩展接口 | 提供规划内容的定义、运行、调试的扩展接口入口 |
| 全流程管理 | 从市场调研、客户开发、项目开发、投产管理、订单管理、生产计划、物料需求计划、采购计划、到料、生产执行、检验、入库、储存、交付、发运、客户账款回款等全流程 |
| 数据输入 | 流程、数据输入、状态明细、增加、删除、变更的内容 |
| 进度查看 | 能够看到全流程业务推进进度，包括所有业务细化子流程 |
| 数据查询 | 对全业务流程数据和记录明细可查，对数据可进行分类汇总 |
| 统计分析 | 可根据需求拉取各种统计数据，并形成各种图表 |
| 问题收集分析 | 对全流程的数据、问题进行收集、分析，找出运营过程中存在的异常 |
| 闭环监控 | 进行改进，能够看到所有异常改进进度，进行闭环监控 |
| 看板管理 | 全流程各节点的运营、管理形成各种看板，含KPI及其他统计图表 |

### 2.3 IT化实现

| 功能名称 | 具体功能要求 |
|---------|-------------|
| 财务管理模块 | 前端模块包含财务管理，实现看异常处理进度和全部功能 |
| 经营管理模块 | 前端模块包含经营管理，实现数据查询、分析及处理、看板管理 |
| 子模块管理 | 客户管理、订单管理、项目管理、采购管理、到料管理、物料管理、生产管理、工艺管理、设备管理、质量管理、仓储管理、交付管理、发运管理等子流程模块前后端全面功能 |
| 财务数据查看 | 所有产品价格、收入明细、已收/待收货款、现金流量表、资产负债表、利润表 |
| 多周期数据 | 财务数据日、周、月、季度、年的数据都有 |
| 数据关联 | 财务管理、经营管理与各子模块的数据从底层进行关联 |
| ERP接入 | 如能通过API则通过API接入ERP各项功能，否则建立模块或通过聊天输入数据或流程文件 |
| Web界面 | 财务管理、经营管理各子模块都有网址（可能集成在一起）、端口并在Docker上运行 |
| 自我进化 | 企业经营、运营、管理水平和功能页面的操作水平的提高以及问题解决能力会自我进化 |
| RAG关联 | 企业经营、运营、管理与RAG、知识图谱关联，实际运行过程中也检索RAG等，形成改进建议与意见 |
| 交互功能 | 企业经营、运营、管理功能与聊天中心关联，可通过聊天功能与内容创作形成交互 |

## 共性要求

所有功能模块必须满足以下要求：

1. **与RAG关联**：所有模块与RAG、知识图谱关联，实际运行时检索RAG
2. **交互功能**：所有模块与聊天中心关联，可通过聊天功能形成交互
3. **Web界面**：所有模块形成前端操作界面，有网址、端口并通过Docker上运行
4. **自我进化**：所有模块具备自我学习和自我进化功能
5. **数据预处理**：所有数据经过预处理、防伪处理后才进入系统
6. **效果反思**：所有模块定期进行效果反思和改进
7. **问题弹窗**：运行中的问题通过弹窗与用户交互
8. **资源适配**：自适应调整电脑资源使用
